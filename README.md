# 🎸 SyncChord — 유튜브 실시간 코드 + 싱크 합주

> 유튜브 영상 위에 실시간 코드 오버레이 + 멀티유저 메트로놈 싱크.
> "같이 치자" 링크 하나면 온라인 합주 시작.

## 핵심 기능
- 유튜브 재생에 맞춰 실시간 코드 표시 (Am, G, C, F...)
- Rust WASM으로 브라우저에서 직접 오디오 분석 (서버 불필요)
- 멀티유저 메트로놈 싱크 (WebSocket)
- 크롬 익스텐션으로 유튜브 위에 오버레이

## 왜 만들었나
Chordify는 좋은데 **유튜브랑 동시에 못 봄**. 탭 왔다갔다 하면서 기타 치기 불편함.
SyncChord는 유튜브 영상 위에 코드가 바로 뜨고, 친구랑 링크 공유하면 같은 메트로놈으로 합주 가능.

```
┌─────────────────────────────────────────┐

└─────────────────────────────────────────┘
         │                    │
         ▼                    ▼
┌─────────────────┐  ┌──────────────────┐
```

## 기술 스택
- **Rust + WASM**: 실시간 오디오 분석 (rustfft, symphonia)
- **Chrome Extension**: 유튜브 오버레이 UI (content script)
- **WebSocket**: 멀티유저 싱크 (tokio + tungstenite 또는 Node.js)
- **코드 DB 폴백**: 인기곡은 DB에서 정확한 타임스탬프 코드 로드

## 코드 분석 전략
```
곡 인식 (fingerprint)
    ↓
코드 DB에 있음? ──→ 타임스탬프 코드 로드 → 유튜브 싱크
    ↓ 없음
Rust WASM 실시간 크로마 분석 → 폴백
```

## 디렉토리 구조
```
SyncChord/
├── rust-wasm/          # Rust 오디오 분석 엔진 (→ WASM 빌드)
│   ├── Cargo.toml
│   └── src/
│       ├── lib.rs      # WASM 진입점
│       ├── chroma.rs   # 크로마그램 계산
│       ├── chord.rs    # 코드 추정 (템플릿 매칭)
│       └── beat.rs     # BPM 감지
├── extension/          # 크롬 익스텐션
│   ├── manifest.json
│   └── src/
│       ├── content.js  # 유튜브 페이지 오버레이
│       ├── popup.js    # 팝업 UI
│       └── sync.js     # WebSocket 싱크 클라이언트
├── server/             # 싱크 서버 (WebSocket)
│   └── index.js        # 방 관리 + 메트로놈 동기화
├── docs/               # 설계 문서
└── README.md
```
